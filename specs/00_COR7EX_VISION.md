# Cor7ex — Hive Mind Plattform

## Was ist Cor7ex?

Cor7ex ist eine **Plattform, auf der sich Menschen und KI-Agenten treffen**, um gemeinsam zu arbeiten, sich auszutauschen und Entscheidungen für das Unternehmen zu treffen.

Cor7ex ist nicht "noch ein Chat", nicht "noch ein RAG-Tool", nicht nur eine Wissensdatenbank — es ist die **zentrale Intelligenz-Einheit** des Unternehmens.

---

## Der Hive Mind Gedanke

**Hive Mind = viele Individuen → ein gemeinsames Denken.**

Der Begriff kommt aus der Natur: Bei Bienen, Ameisen oder Termiten arbeitet der ganze Staat wie ein Organismus. Keine einzelne Einheit "weiß alles", aber zusammen entstehen komplexe Entscheidungen.

### Auf Unternehmensebene bedeutet das:

- **Gemeinsame Ziele, Werte, Strategieverständnis** — Alle ziehen in die gleiche Richtung
- **Schnelle Entscheidungen** — Weniger Abstimmung nötig, hohe Effizienz
- **Implizites Verständnis** — "Man weiß einfach, wie es läuft"
- **Kollektive Intelligenz** — Mensch + KI + Daten + Software ergeben zusammen mehr als die Summe der Teile

### Gesunder Hive Mind

Ein gesunder Hive Mind bedeutet: Gemeinsame Richtung, aber **Raum für Widerspruch**. Vielfalt bleibt erhalten. Gegenmechanismen wie aktive Gegenmeinungen, Devil's Advocate und externe Perspektiven verhindern Groupthink.

## Wie Mensch und KI zusammenarbeiten

### Ein Kanal: Agent-Konversationen

Cor7ex hat **einen einzigen Kanal** für die Zusammenarbeit zwischen Mensch und KI — die Agent-Konversation. Es gibt keinen separaten "Chat" und keinen separaten "Agent-Modus". Jede Interaktion ist eine Konversation mit dem Agenten.

Der Agent ist intelligent genug, um selbst zu entscheiden was nötig ist:

- **Einfache Frage?** → Der Agent antwortet direkt, ohne Tools zu nutzen
- **Frage zum Unternehmenswissen?** → Der Agent durchsucht den Wissensspeicher
- **Komplexe Aufgabe?** → Der Agent nutzt Tools, Skills, Sub-Agenten und arbeitet Schritt für Schritt
- **Mehrstufiger Prozess?** → Der Agent fragt nach, holt Feedback ein, iteriert gemeinsam mit dem Menschen

### Wie fühlt sich das an?

Der Mensch öffnet Cor7ex, startet eine Konversation und beschreibt was er braucht — so wie er es einem kompetenten Kollegen erklären würde. Der Agent versteht die Aufgabe, fragt bei Unklarheiten nach, nutzt die verfügbaren Werkzeuge und liefert Ergebnisse. Der Mensch gibt Feedback, der Agent passt an. So entsteht gemeinsam ein Ergebnis.

Es ist kein Frage-Antwort-Muster. Es ist **Zusammenarbeit** — ein Gespräch zwischen Mensch und KI, bei dem beide Seiten beitragen.

### Das ganze Unternehmen, nicht nur ein User

Cor7ex ist nicht für einen einzelnen Nutzer gebaut — es ist für das **Unternehmen als Organismus**. Jede Abteilung arbeitet mit Cor7ex, und das Wissen fließt zwischen ihnen:

- **Marketing** erarbeitet eine Kampagne mit dem Agent → das Ergebnis wird im Wissensspeicher abgelegt
- **Vertrieb** fragt den Agent nach aktuellen Kampagnen → findet was Marketing erarbeitet hat
- **Geschäftsführung** lässt sich Quartalszahlen auswerten → der Agent kennt Daten aus allen Abteilungen
- **Pflege-Disposition** optimiert Einsatzpläne → die Erfahrungen fließen als Skills zurück in den Schwarm

Jede Konversation, jedes erarbeitete Ergebnis kann zurück in den Wissensspeicher fließen. **Das Unternehmen baut organisch sein eigenes Gedächtnis auf** — durch die tägliche Arbeit aller Abteilungen mit Cor7ex.

### Unternehmensgedächtnis

Was in Konversationen erarbeitet wird, geht nicht verloren. Ergebnisse — Strategiepapiere, Analysen, Reports, Entscheidungen — werden im Wissensspeicher abgelegt und stehen dem gesamten Unternehmen zur Verfügung (nach Berechtigungen). Das geschieht nicht als manueller Extra-Schritt, sondern als natürlicher Teil des Workflows: Skills können am Ende automatisch fragen "Soll ich das Ergebnis im Wissensspeicher ablegen?" oder bei bestimmten Ergebnis-Typen standardmäßig speichern.

### Skills — das mächtigste Werkzeug

Skills sind mehr als vordefinierte Workflows. Sie sind das **kodifizierte Wissen des Unternehmens** — bewährte Prozesse, die von Menschen entwickelt, vom Schwarm bewertet und als Standard übernommen werden.

**Warum Skills so zentral sind:**

- **Wissen wird ausführbar**: Statt einer Prozessbeschreibung in einem PDF wird der Prozess zu einem Skill, den der Agent Schritt für Schritt ausführen kann.
- **Best Practices verbreiten sich organisch**: Ein Mitarbeiter findet einen guten Weg einen Vertrag zu analysieren → erstellt einen Skill → Team übernimmt → wird Schwarm-Standard → das gesamte Unternehmen profitiert.
- **Unternehmensgedächtnis wächst**: Skills sorgen dafür, dass erarbeitete Ergebnisse gespeichert werden und das Wissen im Unternehmen bleibt.
- **Qualität durch den Schwarm**: Schlechte Skills werden herabgestuft, gute werden zum Standard. Keine zentrale Kontrolle nötig.
- **Onboarding wird trivial**: Neue Mitarbeiter haben ab Tag 1 Zugriff auf alle Schwarm-Skills — das gesammelte Wissen des Unternehmens in ausführbarer Form.

**Beispiele für Skills im Unternehmen:**

- "Quartalsreport erstellen" — Daten sammeln, analysieren, Report formatieren
- "Vertrag prüfen" — Risiken identifizieren, Fristen markieren, Zusammenfassung erstellen
- "Neue Werbekampagne" — Zielgruppe definieren, Kanäle vorschlagen, Budget planen
- "Einsatzplan optimieren" — Verfügbarkeiten prüfen, Qualifikationen matchen, Plan erstellen
- "Wettbewerbsanalyse" — Markt recherchieren, Stärken/Schwächen auflisten, Empfehlungen geben
- "Mitarbeiter onboarden" — Checkliste durchgehen, relevante Dokumente zusammenstellen, Schulungsplan erstellen

### Reale Anwendungsfälle

Alles was Führung, Leitung und tägliche Arbeit im Unternehmen betrifft:

### Säule 1: Wissensspeicher — Das Gedächtnis des Unternehmens

Alles was das Unternehmen weiß, an einem Ort — zugänglich für Menschen und Agenten gleichermaßen.

- **Dokumente**: Verträge, Strategiepapiere, Berichte, Handbücher, Präsentationen
- **Unternehmenswissen**: Vision, Werte, Marketingstrategien, Brand Guidelines
- **Verstreute Daten**: Alles was in einer Firma anfällt und heute in verschiedenen Ordnern, E-Mails und Köpfen steckt
- **Intelligente Suche**: Cor7ex versteht Fragen, nicht nur Stichworte. Es findet relevante Informationen auch wenn sie anders formuliert sind als die Frage.

### Säule 2: Agent — Der KI-Arbeitskollege

Der Agent ist die KI-Seite der Zusammenarbeit. Der Mensch beschreibt was er braucht, der Agent versteht, fragt nach, nutzt Werkzeuge und liefert Ergebnisse.

- **Konversationen**: Mensch und Agent arbeiten gemeinsam in einem Gespräch. Der Agent fragt nach, der Mensch gibt Feedback, zusammen entsteht das Ergebnis.
- **Skills**: Bewährte Arbeitsabläufe die der Schwarm entwickelt hat. Z.B. "Quartalsreport erstellen" oder "Vertrag prüfen". Skills machen das Wissen des Unternehmens ausführbar.
- **Werkzeuge**: Der Agent kann im Wissensspeicher suchen, Dokumente erstellen, Daten auswerten, Benachrichtigungen senden — und über Enterprise-Anbindungen auch in der bestehenden Software des Unternehmens handeln.
- **Delegation**: Für komplexe Aufgaben kann der Agent Spezialisten starten, die Teilaufgaben übernehmen.
- **Im Auftrag des Menschen**: Agenten handeln nie eigenständig. Sie handeln immer im Auftrag und mit den Berechtigungen des Menschen der sie beauftragt.

### Säule 3: Enterprise-Anbindungen — Die Software des Unternehmens

Cor7ex steht nicht alleine. Es verbindet sich mit der bestehenden Software-Landschaft des Unternehmens.

- **Interne Systeme**: Einsatzplanung, Zeiterfassung, Buchhaltung — alles was das Unternehmen bereits nutzt
- **Externe Plattformen**: Dienste und Systeme außerhalb des Unternehmens
- **Prinzip**: Jedes System das angebunden wird, wird zu einem Werkzeug das der Agent nutzen kann
- **Ziel**: Der Agent kann nicht nur Wissen finden, sondern auch in Systemen handeln — Pläne erstellen, Daten abrufen, Prozesse anstoßen

## Der Schwarm-Gedanke im Skill-System

Skills verkörpern den Hive Mind auf technischer Ebene:

```
Bienenschwarm:                        Cor7ex:
Spurbiene findet guten Nistplatz  →   Mitarbeiter erstellt nützlichen Skill
Tanzt intensiver je besser        →   Skill wird genutzt + positiv bewertet
Andere Bienen folgen dem Tanz     →   Team-Mitglieder übernehmen den Skill
Schwarm entscheidet ohne Chef     →   Schwarm-Adoption durch Usage + Votes
```

**Kein Gatekeeper, der Schwarm entscheidet.**

- **Personal**: Jeder kann Skills erstellen und experimentieren
- **Team**: Teilen mit der Abteilung, Feedback durch Votes
- **Schwarm**: Automatische Promotion zum Unternehmens-Standard bei genug Adoption + positiver Bewertung

---

## Leitprinzipien

1. **Hive Mind First** — Jede Feature-Entscheidung wird daran gemessen: Fördert es die kollektive Intelligenz aus Mensch + KI?

2. **Schrittweise Anbindung** — Kein Big Bang. Jedes neue System wird ein Tool. Nach und nach aufbauen.

3. **Der Schwarm entscheidet** — Kein Gatekeeper für Skills. Die Nutzung und Bewertung durch die Community bestimmt was Standard wird.

4. **Agenten handeln im Auftrag** — KI-Agenten sind keine eigenständigen Akteure. Sie handeln mit den Rechten und im Auftrag von Menschen.

5. **Transparenz** — Jede KI-Aktion ist nachvollziehbar: welches Model, welche Tools, welche Quellen, welche Kosten.
